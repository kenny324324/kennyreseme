設計流程｜Figma

**需求收集與競品分析：**公車處提供功能需求並提供市面上公車 App 作為參考，進行競品分析與功能對比。

**多版本設計提案：**根據需求設計三種不同架構的介面方案，經客戶評估後選定第二版作為開發基礎。

**視覺風格確立：**依據客戶需求採用##暖色系##作為主色調，最終選定**暖橘棕色**營造友善親切的使用體驗。

**設計系統建立：**基於選定方案建立完整的 UI 元件庫，確保跨頁面的視覺一致性。

開發流程｜技術

**技術選定：**依據公司技術規範，採用**.NET MAUI**作為跨平台開發框架，整合**Firebase**推播服務與**Google Maps API**地圖功能。

**開發分工協作：**負責**UI/UX 規劃**與**APP 端開發**，包含 API 串接與資料呈現邏輯，與後端工程師協作定義所需資料格式與 API 規格。

**核心技術挑戰：**##Firebase 推播整合##為最具挑戰性的技術點，另外##地圖功能的流程設計與介面實現##也投入大量時間進行優化調整。

**專案管理流程：**透過 PM 進行需求溝通與問題回報，採用敏捷開發模式，即時響應需求變更並發布更新版本。

**專案時程：**目前專案進行**兩年多**，已進入第二階段開發，持續進行功能維護與優化。
