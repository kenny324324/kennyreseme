挑戰與解決 #1：Firebase 推播整合

**情境：**首次導入 Firebase 推播功能，需要讓客戶能在後台成功發送通知到使用者手機上。

**困難點：**##全新技術領域##，但過去沒有相關經驗。主要挑戰包括透過 API 儲存 Firebase 推播 token、建立後台與使用者裝置的通訊機制，以及確保推播通知能穩定送達。

**解決過程：**採用**持續研究與測試**的方式，深入研讀 Firebase 官方文檔，與後端工程師密切協作討論每個使用者帳號的推播 token 存取方式。透過**不斷測試與調整**，逐步建立穩定的推播機制。

**時程與成果：**歷時**數個月**完成整合，最終成功建立完整的推播系統，讓客戶能透過後台即時發送通知，大幅提升使用者互動體驗。

挑戰與解決 #2：地圖與公車站點整合

**情境：**客戶要求在地圖上實現點擊功能，能標示附近公車站點供使用者選擇，提升路線規劃的便利性。

**困難點：**單純顯示地圖並不困難，但要讓**後端資料與地圖連動**是主要挑戰。需要處理地圖點擊事件、計算附近站點、動態標記顯示等複雜邏輯。

**解決過程：**透過**不斷嘗試與查資料**，研究 地圖 API 的最佳實作方式。測試多種資料連動方案，最終找出最適合的顯示方式，確保使用者體驗流暢。

**成果：**成功實現地圖點擊標示功能，使用者可以直觀地在地圖上選擇公車站點，大幅簡化路線規劃流程。
